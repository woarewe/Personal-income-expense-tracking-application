## Название прецедента: 
Регистрация
## Действующее лицо:
Гость
## Цель: 
Получение нового аккаунта
## Предусловие: 
Пользователь посетил страницу регистрации
## Главная последовательность:
1. Система направляет пользователя на страницу входа.
2. Пользователь заполняет форму и нажимает кнопку 'Регистрация'.
3. Система сохраняет данные пользователя и перенаправляет его на страницу аккаунта.
##  Альтернативная последовательность(введены некоректные данные):
1. Система выводит предупреждение о некорректных данных.
2. Пользователю предоставляется повторная попытка для ввода данных.
## Постусловие: 
Пользователь получает доступ к новому аккаунту.

## Название прецедента: 
Создание категории
## Действующее лицо:
Зарегистрированный пользователь
## Цель: 
Создание категории с определнным названием и типом
## Предусловие: 
Пользователь находится на странице аккаунта.
## Главная последовательность:
1. Пользователь нажимает на кпопку 'Добавить категорию'
2. Cистема перенаправляет пользователя на страницу с формой
3. Пользователь вводит название категории и её тип и нажимает кпопку 'Создать'
4. Введены корректные данные.
5. Cистема сохраняет категорию с полученными данными
6. Пользователь перенаправляется на свою страницу и получает уведомлени об успешной операции.
##  Альтернативная последовательность(введены некорректные данные):
1. Введеные некоретные данные.
2. Пользователь получает уведомление от системы о некорректных данных.
3. Пользователю предоставляется повторная попытка для ввода данных
## Постусловие: 
К аккаунту пользователя привязывается новая категория.

## Название прецедента: 
Создание транзакции
## Действующее лицо:
Пользователь
## Цель: 
Создание транзакции с предположительной датой события, её значения и прикрепления её к определенной категории.
## Предусловие: 
Пользователь находится на странице своего аккаунта данного сайта.
## Главная последовательность:
1. Пользователь нажимает на кпопку 'Добавить транзакцию'
2. Cистема перенаправляет пользователя на страницу с формой
3. Пользователь выбирает категорию, предположительную дату, размер транзакции(в денежном эквиваленте) и нажимает кнопку создать.
4. Введены корректные данные.
5. Cистема сохраняет транзакцию с полученными данными
6. Пользователь перенаправляется на свою страницу и получает уведомлени об успешной операции.
##  Альтернативная последовательность(введены некорректные данные):
1. Введены некорректные данные.
2. Пользователь получает предупреждение о том, что введеные некорректные данные.
3. Пользователю предоставляется возможность для повтороно введение данных.

## Постусловие: 
У пользователя появляется инормация о транзакции, привзяанной к определенной категории.
